🔬 Cell Validator — инструкция

Как загружать JSON, размечать клетки, сохранять прогресс и экспортировать отчёт

Открыть инструмент

1) Быстрый старт

  1. Откройте index.html и введите имя валидатора.
  2. Загрузите данные одним из способов:
    • 📂 Загрузить JSON — один файл;
    • 📁 Папка annotations — выбрать папку с несколькими JSON.
  3. Выберите файл из списка доступных и начните валидацию.

2) Как размечать клетки

  • Текущая клетка показана справа (увеличенный фрагмент).
  • Ставьте метку кнопками в блоке «Назначить метку» или горячими клавишами: 1 steatosis, 2 mesenchymal_cells, 3 normal, 4 non_nuclear, 5 balloon_dystrophy.
  • После выбора метки инструмент автоматически переходит к следующей клетке.
  • Навигация: ←/→ — предыдущая/следующая, кнопка «К последней» — к последней размеченной, поле «№» — перейти к конкретной клетке.

3) Сохранение прогресса

Локальное автосохранение

Прогресс сохраняется автоматически в браузере и восстанавливается при повторном входе под тем же именем валидатора.

Сохранение в файл (рекомендуется)

  1. Нажмите 💾 Папка сохранения.
  2. Выберите папку для прогресса (можно рядом с проектом).
  3. Инструмент будет автоматически обновлять файл вида: <имя_json>__<валидатор>.validation.json.
Если работаете через file://, получение списка файлов с сервера недоступно — используйте кнопку 📁 Папка annotations.

4) Экспорт результата

Кнопка 📊 Экспорт Excel формирует общий отчёт по всем размеченным файлам текущего валидатора.

В таблицу входят: имя валидатора, файл, индекс клетки, координаты контура, исходная метка, метка валидатора, тип формы.

Перед завершением работы всегда делайте экспорт Excel как резервную копию.

5) Возможные проблемы

  • Не открывается список JSON — проверьте, что выбрана папка с файлами .json.
  • Нет изображения, только контуры — в JSON отсутствует imageData.
  • Прогресс не восстановился — убедитесь, что имя валидатора введено точно так же, как раньше.
